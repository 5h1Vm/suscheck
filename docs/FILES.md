# SusCheck: Project Structure & File Index

This document provides a comprehensive guide to every folder and file in the SusCheck repository. The project is organized to separate core logic, pluggable security modules, and comprehensive testing artifacts.

## 📂 src/suscheck/
The heart of the platform, structured for maximum modularity.

### **Core Engine (`core/`)**
*   `auto_detector.py`: Hardened artifact identification (Magic bytes, shebangs, heuristics).
*   `pipeline.py`: The 3-tier orchestration engine for all scanners.
*   `risk_aggregator.py`: The 10-step PRI scoring algorithm and verdict logic.
*   `config_manager.py`: Centralized, `.env`-first configuration handler.
*   `finding.py`: Unified data model for security findings.
*   `diagnostics.py`: Self-test and key-verification logic.

### **Security Modules (`modules/`)**
*   `code/`: Tier 1 (Static) and Tier 2 (Semgrep/Bandit) code scanners.
*   `config/`: Infrastructure-as-Code (IaC) auditing via Checkov.
*   `repo/`: Repository-level analytics and trust scoring.
*   `mcp/`: Specialized auditing for Model Context Protocol servers.
*   `reporting/`: Terminal and JSON reporting formatters.

### **AI & Logic (`ai/`)**
*   `triage_engine.py`: Multi-model AI triage for false-positive rejection.
*   `explain_engine.py`: Logic for interactive vulnerability walkthroughs (V2 Roadmap).
*   `key_resolution.py`: Handles secure API key discovery in multi-environment setups.
